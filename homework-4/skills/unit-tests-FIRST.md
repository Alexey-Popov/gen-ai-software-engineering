# Skill: Unit Tests FIRST

This skill defines the **FIRST** principles that every test generated by the Unit Test Generator must satisfy. Each generated test class must include a `// FIRST: F✓ I✓ R✓ S✓ T✓` comment block at the top explaining compliance or any partial exception.

---

## The FIRST Principles

| Letter | Principle | Requirement |
|--------|-----------|-------------|
| **F** | Fast | Each test completes in < 100 ms. No file I/O, no network calls, no `Thread.Sleep`. |
| **I** | Independent | Tests do not share mutable state. No ordering dependency. Each test sets up its own data and tears down cleanly. |
| **R** | Repeatable | The same result is produced on every run. All non-determinism (clocks, GUIDs, randomness) is mocked or seeded. |
| **S** | Self-validating | Every test contains at least one FluentAssertions `.Should()` assertion. No `Console.WriteLine` as the only verification. |
| **T** | Timely | Tests are written only for code changed by the Bug Fixer in this pipeline run. Do not generate tests for unchanged code. |

---

## How to Apply

1. For each changed method or class identified in `fix-summary.md`, generate one or more tests.
2. Before writing each test, check it against all five FIRST letters.
3. At the top of each generated test class, add a comment block:

```csharp
// FIRST compliance:
// F – Fast:        No I/O or sleep; all dependencies mocked.
// I – Independent: Each test creates its own mock and data; no shared state.
// R – Repeatable:  All GUIDs seeded; DateTime mocked via fixed value.
// S – Self-validating: Every test has a FluentAssertions .Should() call.
// T – Timely:      Tests cover only code changed in this pipeline run.
```

4. If a principle cannot be fully satisfied, note the exception and reason in the comment block (e.g. `R – partial: integration test requires real in-memory host`).

---

## Test Framework Conventions (this project)

- **Framework**: NUnit 4
- **Assertions**: FluentAssertions
- **Mocking**: Moq
- **Integration tests**: `WebApplicationFactory<Program>` with a real in-memory host
- **Unit tests**: mock `ITicketRepository` and `IClassificationService` directly
- **Naming**: `MethodName_Scenario_ExpectedResult` (e.g. `TransitionTo_FromNewToInProgress_Succeeds`)
- **Test command**: `dotnet test` from the repository root
